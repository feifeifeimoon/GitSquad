# Contributing to GitSquad

## Quick Start

```bash
# 1. Clone & install
git clone https://github.com/feifeifeimoon/GitSquad.git
cd GitSquad

# 2. Start PostgreSQL
createdb gitsquad

# 3. Configure environment
cp .env.example .env
# Edit .env: fill in GITSQUAD_GOOGLE_CLIENT_ID and GITSQUAD_GOOGLE_CLIENT_SECRET
# Register at https://console.cloud.google.com/apis/credentials
#   Authorized redirect URI: http://localhost:8080/api/v1/auth/google/callback

# 4. Start backend
source .env
go run ./cmd/server
# → http://localhost:8080

# 5. Start frontend
cd web && npm run dev
# → http://localhost:3000
```

**Layers**: `handler → service → store → (sqlc generated db)`

- **handler** — HTTP parsing, response formatting. No business logic.
- **service** — Business logic, orchestration. Calls store methods. Zero SQL.
- **store** — Type-safe data access generated by sqlc from `.sql` files.
- **types** — Pure struct definitions, no methods.

## Database

### Postgres Setup

```bash
createdb gitsquad
export GITSQUAD_DATABASE_URL=postgres://postgres:postgres@localhost:5432/gitsquad?sslmode=disable
```

Tables are created automatically on server start via `database/migration.go`.

### Adding a new query

1. Write SQL in `store/queries/<entity>.sql`:

```sql
-- name: FindByProvider :one
SELECT * FROM users WHERE id = $1;
```

2. Regenerate Go code:

```bash
sqlc generate
```

3. Call from service:

```go
user, err := s.store.FindByProvider(ctx, id)
```

### Adding a new table

1. Add `CREATE TABLE` to `database/migration.go` (for auto-migration)
2. Add `CREATE TABLE` to `store/schema.sql` (for sqlc type generation)
3. Add queries to `store/queries/`
4. Run `sqlc generate`

### sqlc annotations

| Annotation | Purpose |
|-----------|---------|
| `:one` | Returns a single row |
| `:many` | Returns multiple rows |
| `:exec` | INSERT / UPDATE / DELETE with no return |
| `:execrows` | Same as exec, returns affected row count |

## Development Workflow

### Run locally

```bash
# Terminal 1: Backend
source .env && go run ./cmd/server

# Terminal 2: Frontend
cd web && npm run dev

# Terminal 3: CLI (optional)
go run ./cmd/gitsquad daemon login
```

### Before committing

```bash
# Build everything
go build ./cmd/server/...
go build ./cmd/gitsquad/...
cd web && npx next build

# Regenerate sqlc (if SQL changed)
sqlc generate
```

### Environment variables

| Variable | Required | Default |
|----------|----------|---------|
| `GITSQUAD_DATABASE_URL` | Yes | — |
| `GITSQUAD_GOOGLE_CLIENT_ID` | Yes | — |
| `GITSQUAD_GOOGLE_CLIENT_SECRET` | Yes | — |
| `GITSQUAD_HTTP_ADDR` | No | `:8080` |
| `GITSQUAD_ENV` | No | `development` |
| `GITSQUAD_GOOGLE_CALLBACK_URL` | No | `http://localhost:8080/api/v1/auth/google/callback` |
| `GITSQUAD_JWT_SECRET` | No | `gitsquad-dev-secret` |
| `GITSQUAD_FRONTEND_URL` | No | `http://localhost:3000` |
| `GITSQUAD_LOG_LEVEL` | No | `info` (set to `debug` for verbose) |

### CLI

```bash
# Authenticate this machine
gitsquad daemon login                        # Browser pairing
gitsquad daemon login --token gtsq_dm_xxxxx  # Headless / CI

# Scan for AI CLI tools
gitsquad daemon status

# Start the daemon
gitsquad daemon run
```

## API Design

All API responses use a standard envelope:

```json
{
  "success": true,
  "data": {},
  "message": "optional error message"
}
```

Endpoints use structured logging (`log/slog`) with JSON output. Logs include method, path, status, and latency.
